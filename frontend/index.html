为什么需要 AgentPass

AI Agent 的身份危机

当 AI Agent 代表人类操作企业系统时，谁来验证它的身份？谁来约束它的权限？谁来记录它的每一步行为？

🔐

零信任 Agent 身份

每个 Agent 持有 SPIFFE 风格的 SVID 证书和 RSA 密钥对，mTLS 签名绑定每次 API 调用，杜绝 Token 盗用与身份冒充。

⚖️

动态最小权限

基于任务意图实时推断所需能力，以用户权限 ∩ Agent 能力 ∩ 任务意图三者交集签发 Token，权限在委托链中单调递减。

⛓️

不可篡改审计链

每条审计日志包含前一条的 SHA-256 哈希，形成类区块链的完整性保证。任何篡改行为都会在验证时被立即发现。

12 项核心创新

从标准到突破

每一项创新都对应一个真实的 AI 安全威胁，从协议设计到实现细节，全面覆盖 AI Agent 生命周期安全。

01 / 12

Agent Card 能力声明标准

参考 Google A2A 协议，为每个 Agent 生成结构化 JSON 能力名片。委托前强制能力比对，不匹配直接在 Token 签发阶段拒绝。

Google A2A
02 / 12

衰减式 Token 链

参考 IETF draft，每次 Token 向下传递时权限范围单调收缩，三方交集计算，Token 链记录衰减层级与权限上界 max_scope。

IETF Draft
03 / 12

跨 Agent 特权升级防御

参考 Johann Rehberger 2025 年公开的 Cross-Agent 攻击手法，实现 PrivilegeEscalationDetector，检测到异常立即撤销所有活跃 Token。

A07 OWASP
04 / 12

Agent 会话走私防御

参考 Palo Alto Unit 42 研究，对每个 A2A 会话绑定不可伪造指纹（时间戳 + 调用链哈希 + 随机 nonce），验证连续性。

Session Security
05 / 12

人工审批熔断机制

涉及敏感操作自动挂起为 PENDING_HUMAN_APPROVAL，前端实时弹窗，30 秒超时自动拒绝。审批通过后签发 5 分钟单次使用 Token。

Human-in-Loop
06 / 12

实时五维风险评分

每次授权决策计算 0-100 风险分：请求频率、链深度、时间窗口、能力组合危险度、历史违规。70+ 降级只读，90+ 冻结 Agent。

Real-time
07 / 12

Prompt Injection 语义防御

三层检测：关键词匹配 + 正则模式 + 语义规则，覆盖系统提示词覆盖、权限声明注入、未授权工具调用等攻击向量。

A01 OWASP
08 / 12

不可变审计日志链

每条日志含 prev_log_hash（前一条 SHA-256），形成类区块链结构。提供 verify_audit_chain 接口实时验证完整性，篡改立即可检。

A10 OWASP
09 / 12

Agent 行为基线与异常检测

前 10 次调用自动建立行为基线，偏差 >2σ 触发 BEHAVIOR_ANOMALY 告警，>3σ 触发 Token 撤销。基线数据持久化 SQLite。

Statistical
10 / 12

动态最小权限计算

分析任务意图文本，使用关键词提取识别本次任务所需最小能力集。Token scope 始终为三方交集，而非静态配置。

A05 OWASP
11 / 12

零信任 mTLS Agent 身份绑定

参考 NIST SP 800-207A，每个 Agent 持有 RSA-2048 密钥对，每次 A2A 调用用私钥签名请求体，服务端用注册公钥验签。

NIST 800-207A
12 / 12

OPA 风格策略即代码引擎

纯 Python 实现轻量级策略引擎，从 YAML 加载策略，支持时间窗口、风险分阈值、委托深度等条件，deny 优先于 allow，支持热重载。

Policy-as-Code
